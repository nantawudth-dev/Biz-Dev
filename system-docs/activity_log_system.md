# ระบบบันทึกกิจกรรม (Activity Logging System)

เอกสารนี้อธิบายการทำงานของระบบบันทึกกิจกรรม (Activity Logs) ในระบบ Biz-Dev ซึ่งมีเป้าหมายเพื่อใช้ในการตรวจสอบความถูกต้อง (Auditing) ติดตามประวัติการใช้งานระบบ และรักษาความปลอดภัยข้อมูล

## 📌 โครงสร้างฐานข้อมูล
ระบบใช้ตาราง `activity_logs` ในฐานข้อมูล Supabase เพื่อเก็บประวัติ ประกอบด้วยฟิลด์สำคัญดังนี้:
*   `id` (uuid): รหัสประจำระเบียนบันทึก
*   `user_id` (uuid): รหัสผู้ใช้งานที่กระทำการ (อ้างอิงจากตาราง users)
*   `user_email` (text): อีเมลของผู้ใช้งาน (เพื่อความสะดวกในการค้นหาและแสดงผล)
*   `action` (text): ประเภทของการกระทำ เช่น `CREATE`, `UPDATE`, `DELETE`, `LOGIN`, หรือประเภทเจาะจงอื่นๆ
*   `entity_type` (text): ประเภทของข้อมูลที่ถูกกระทำ เช่น `entrepreneur`, `project`, `consultant`, `course`, `user`, `establishment_type`, `business_category`
*   `entity_id` (text): รหัสหรือ ID ของข้อมูลที่เกี่ยวข้อง (เพื่อใช้เชื่อมโยงกลับไปยังข้อมูลนั้นๆ)
*   `entity_name` (text): ชื่อหรือ Title ของข้อมูล (เพื่อง่ายต่อการอ่านว่าแก้ไขเรื่องอะไรไป)
*   `details` (jsonb): ข้อมูลรายละเอียดเพิ่มเติม หรือ Snapshot ของการแก้ไข (เช่น ข้อมูลก่อน-หลังแก้ไข)
*   `created_at` (timestamptz): วันเวลาที่เกิดกิจกรรมนั้นๆ

## ⚙️ กระบวนการทำงานแบบ Real-time
โปรแกรมฝั่งหน้าบ้านของแอดมินหรือระบบ จะเรียกใช้ฟังก์ชัน `logActivity` จากไฟล์ `services/dataService.ts` เพื่อบันทึกประวัติล่วงหลังการทำงานสำเร็จ (เช่น หลังจากสั่ง Insert หรือ Update สำเร็จในบรรทัดถัดไปจะเรียกบันทึกกิจกรรม)

### ตัวอย่างสิ่งที่ระบบตรวจสอบและติดตาม (Tracked Entities)
- ข้อมูลผู้ประกอบการ (Entrepreneur)
- ข้อมูลโครงการ (Project)
- ข้อมูลที่ปรึกษา (Consultant)
- หลักสูตรอบรม (Course)
- การจัดการผู้ใช้งานระบบ (User Management)
- ข้อมูล Master Data พื้นฐาน (Settings: หมวดหมู่ธุรกิจ, ประเภทองค์กร ฯลฯ)

## 📊 การแสดงผลบนหน้าจอ (UI Layer)
ระบบจัดการการแสดงผลบันทึกประวัติไว้ที่เมนู **Settings -> Activity Logs (แท็บบันทึกกิจกรรม)**
โดยมีความสามารถดังนี้:
1.  **Search & Filter:** สามารถพิมพ์ค้นหาตามชื่อ/อีเมลผู้กระทำ ค้นหาตาม "ประเภทการกระทำ" (Action) ตาม "เป้าหมายที่กระทำ" (Entity Type) และช่วงวัน-เวลาแบบกำหนดเองได้
2.  **Pagination:** รองรับการแบ่งหน้าการแสดงผล และกำหนดจำนวนรายการต่อหน้าได้ (เช่น 10, 20 หรือ 50 รายการ/หน้า) เพื่อไม่ให้โหลดข้อมูลมาแสดงพร้อมกันหนักเกินไปในครั้งแรก
3.  **Viewing Details:** รายการใดที่มีข้อมูลแบบ JSON ซับซ้อน จะมีปุ่มเปิดดู Snapshot รายละเอียดได้

## 🗑️ ระบบทำความสะอาดอัตโนมัติ (Auto-Cleanup / Data Retention)
เนื่องจากตารางบันทึกกิจกรรมจะมีขนาดใหญ่และกินจำนวนแถว (Rows) ของ Database รวดเร็วที่สุด จึงมีการออกแบบ **นโยบายการจัดเก็บข้อมูล (Retention Policy) ให้อยู่ที่ 90 วัน**

ระบบจะทำงานร่วมกับ **`pg_cron` Scheduled Job** ที่ถูกติดตั้งตั้งค่าการตั้งเวลาไว้บนเซิร์ฟเวอร์ PostgreSQL ของ Supabase เอง:

*   **กำหนดการรัน (Schedule):** เที่ยงคืนของเวลาสากล (`0 0 * * *`)
*   **คำสั่งที่ใช้ลบ (Operation):**
    ```sql
    DELETE FROM activity_logs WHERE created_at < NOW() - INTERVAL '90 days';
    ```
ระบบนี้จะทำให้ไม่ต้องเขียน API ควบคุมจากฝั่ง Front-end (React/Next) ไม่ต้องรัน Node Server ทิ้งไว้ แต่ฐานข้อมูลจะรับรู้และมีกลไกดูแลสะสางขยะของตัวเองอย่างชาญฉลาด
